WorkflowEngineInfo#

class WorkflowEngineInfo#

Stores workflow engine information collected by the get_server_info call.

Overview#

release_year

Year portion of the release, such as 2022.

release_id

ID portion of the release, such as 2 for 2022 R2.

build

Build number.

is_release_build

Flag indicating if the release is a production release.

build_type

Build type.

version_as_string

Version of the workflow engine encoded for human consumption.

server_type

Type of server that is responding to the request.

install_location

Installation directory.

base_url

Base URL for clients to connect to.

Import detail#

from ansys.engineeringworkflow.api.datatypes import WorkflowEngineInfo

Attribute detail#

WorkflowEngineInfo.release_year: int#

Year portion of the release, such as 2022.

WorkflowEngineInfo.release_id: int#

ID portion of the release, such as 2 for 2022 R2.

WorkflowEngineInfo.build: int#

Build number.

WorkflowEngineInfo.is_release_build: bool#

Flag indicating if the release is a production release.

The value is False for development, alpha, beta, and other releases.

WorkflowEngineInfo.build_type: str#

Build type.

This attribute must be blank for a production release. For other release types, it may include arbitrary information like the branch that a development build was built from.

WorkflowEngineInfo.version_as_string: str#

Version of the workflow engine encoded for human consumption.

For example, `2022r1 build 333 ALPHA`.

WorkflowEngineInfo.server_type: str#

Type of server that is responding to the request.

The value is a string similar to 'ModelCenter' or 'optiSLang'.

WorkflowEngineInfo.install_location: str | None#

Installation directory.

If the client is on the same box as the workflow engine, this optional attribute may provide the installation folder.

Typically server-based products do not provide this field for security reasons.

WorkflowEngineInfo.base_url: str | None#

Base URL for clients to connect to.

If this is a server-based product ready to receive incoming connections from remote clients, this optional attribute may provide the base URL for clients to connect to.
